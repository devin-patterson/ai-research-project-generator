**1. Key Concepts and Their Definitions**

The topic encompasses various software engineering concepts, which can be categorized into several areas:

* **Architecture Patterns**: These are high-level design patterns that guide the structure and organization of software systems.
	+ Layered Architecture: Separates concerns by dividing the system into layers (e.g., presentation, business logic, data access).
	+ Microservices Architecture: Decomposes the system into small, independent services communicating with each other using APIs.
	+ Event-Driven Architecture: Focuses on producing and consuming events to enable loose coupling between components.
	+ Clean Architecture: Emphasizes separation of concerns, testability, and scalability by separating business logic from infrastructure.
	+ CQRS (Command Query Responsibility Segregation): Separates read and write operations into different models for improved performance and maintainability.
	+ Hexagonal Architecture: Focuses on the application's business logic, isolating it from external dependencies.
* **Distributed Systems Patterns**: These patterns address challenges in distributed systems, such as fault tolerance, scalability, and consistency.
	+ Leader-Follower Pattern: Assigns a leader node to manage the system, with followers providing redundancy.
	+ Sharding Pattern: Distributes data across multiple nodes for improved performance and availability.
	+ Circuit Breaker Pattern: Monitors service calls and detects failures, preventing cascading failures.
	+ Saga Pattern: Manages long-running transactions by breaking them into smaller, compensable units.
	+ 2PC (Two-Phase Commit) Pattern: Ensures atomicity in distributed transactions by dividing the commit process into two phases.
* **GoF Design Patterns**: These patterns provide general solutions to common problems that arise during software design.
	+ Creational Patterns:
		- Singleton Pattern: Ensures a class has only one instance, providing global access.
		- Factory Pattern: Provides an interface for creating objects without specifying their concrete classes.
		- Builder Pattern: Separates construction and representation of complex objects.
		- Prototype Pattern: Creates copies of existing objects by cloning them.
	+ Structural Patterns:
		- Adapter Pattern: Converts the interface of a class to match another's requirements.
		- Bridge Pattern: Decouples abstraction from implementation using separate hierarchies.
		- Composite Pattern: Composes objects into a tree-like structure, enabling polymorphism and flexibility.
		- Decorator Pattern: Dynamically adds behavior to an object without altering its structure.
		- Facade Pattern: Provides a simplified interface for complex systems by hiding internal details.
		- Proxy Pattern: Controls access to an object or resource by providing a surrogate or placeholder.
	+ Behavioral Patterns:
		- Observer Pattern: Allows objects to notify each other about changes.
		- Strategy Pattern: Encapsulates algorithms and allows them to be selected dynamically at runtime.
		- Command Pattern: Represents requests as objects, enabling queuing and logging of commands.
		- State Pattern: Manages object behavior based on its internal state.
		- Iterator Pattern: Allows sequential traversal of aggregate objects without exposing their underlying structure.
* **Algorithm Paradigms**: These provide general frameworks for solving problems using algorithms.
	+ Divide and Conquer: Breaks down complex problems into smaller sub-problems, solving them recursively.
	+ Dynamic Programming: Solves problems by breaking them into overlapping sub-problems, storing solutions to sub-problems in a table.
	+ Greedy Algorithm: Makes locally optimal choices at each step, hoping for an overall optimal solution.
	+ Backtracking Algorithm: Recursively explores possible solutions, backtracking when a dead end is reached.
* **Sorting Algorithms**: These algorithms sort data based on specific criteria, such as numerical or lexicographic order.
	+ Merge Sort: Uses a divide-and-conquer approach to merge sorted sub-arrays into a single sorted array.
	+ Quick Sort: Selects a pivot element and partitions the array around it, recursively sorting sub-arrays.
	+ Heap Sort: Uses a binary heap data structure to sort elements in O(n log n) time.
	+ Tim Sort: A hybrid sorting algorithm combining elements of merge sort and insertion sort for efficient performance.
* **Graph Algorithms**: These algorithms solve problems on graph structures, such as finding paths or minimum spanning trees.
	+ BFS (Breadth-First Search): Explores nodes in a graph level by level, starting from a given source node.
	+ DFS (Depth-First Search): Traverses the graph depth-first, exploring as far as possible along each branch before backtracking.
	+ Dijkstra's Algorithm: Finds the shortest path between two nodes in a weighted graph.
	+ A* (A-Star) Algorithm: An extension of Dijkstra's algorithm using an admissible heuristic to guide search towards the goal node.
	+ Prim's Algorithm: Finds the minimum spanning tree of a connected, undirected graph.
	+ Kruskal's Algorithm: Builds the minimum spanning tree by selecting edges with the lowest weight that do not form cycles.
	+ Topological Sort: Orders vertices in a directed acyclic graph (DAG) such that for every edge (u,v), vertex u comes before v in the ordering.
* **String Algorithms**: These algorithms solve problems related to string manipulation, such as searching or matching patterns.
	+ KMP (Knuth-Morris-Pratt) Algorithm: Searches for a pattern in a text efficiently by precomputing information about the pattern's structure.
	+ Rabin-Karp Algorithm: Uses hashing and the rolling hash technique for efficient pattern matching.
	+ Boyer-Moore Algorithm: A string searching algorithm that uses a combination of bad character heuristics and good suffix heuristics to efficiently find occurrences of a pattern in a text.
